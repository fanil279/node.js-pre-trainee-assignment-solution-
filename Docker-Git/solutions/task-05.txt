Task 05 - Optimizing Docker Images with Multi-stage Builds

Single-stage Dockerfile

FROM node:24-alpine

WORKDIR /app

COPY package*.json ./
RUN npm install

COPY . .

RUN npm run build

EXPOSE 3000

CMD ["npm", "start"]

--------------------------------------------------

Multi-stage Dockerfile

# Builder stage
FROM node:24 AS builder

WORKDIR /app

COPY package*.json ./
RUN npm install

COPY . .

RUN npm run build
RUN npm prune --omit=dev

# Production stage
FROM node:24-alpine AS production

WORKDIR /app

COPY --from=builder /app/package*.json ./
COPY --from=builder /app/node_modules ./node_modules
COPY --from=builder /app/dist ./dist

EXPOSE 3000

CMD ["npm", "start"]

--------------------------------------------------

.dockerignore

.git
node_modules
dist
.gitignore
.prettier
.prettierignore

--------------------------------------------------

Image Size Comparison

Output of docker images

REPOSITORY       TAG      SIZE
typescript-app   single   225 MB
typescript-app   multi    167 MB

The multi-stage Docker image is 58 MB smaller because the final image contains only the compiled application and production
dependencies. TypeScript source files, build tools, and development dependencies are excluded from the production image.

After adding the .dockerignore file, rebuilding the image was slightly faster because Docker sent fewer files to the build context.

--------------------------------------------------

Docker Commands Used

Build the single-stage image:

docker build -f Dockerfile.single -t typescript-app:single .

Build the multi-stage image:

docker build -t typescript-app:multi .

List Docker images:

docker images

Run the single-stage container:

docker run -d -p 3000:3000 --name ts-single typescript-app:single

Test the application:

curl http://localhost:3000

Response:

Hello Docker Multi-stage!

Stop and remove the single-stage container:

docker stop ts-single
docker rm ts-single

Run the multi-stage container:

docker run -d -p 3000:3000 --name ts-multi typescript-app:multi

Test the application:

curl http://localhost:3000

Response:

Hello Docker Multi-stage!

Stop and remove the multi-stage container:

docker stop ts-multi
docker rm ts-multi
