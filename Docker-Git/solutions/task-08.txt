Strategy for Handling Upstream Changes

1. Fetch the latest changes from the upstream repository using:
   git fetch upstream

2. Switch to the local main branch.

3. Merge the latest upstream changes into the local main branch:
   git merge upstream/main

4. Push the updated main branch to the fork repository:
   git push fork main

5. If working on a feature branch, rebase or merge the updated main branch before continuing development to keep the branch up to date.

Best Practices for Commit Messages Learned

- Use clear and descriptive commit messages.
- Follow the imperative mood (e.g., "add", "fix", "update").
- Keep each commit focused on a single logical change.
- Use conventional commit prefixes such as feat:, fix:, docs:, chore:, and refactor:.
- Write small, meaningful commits instead of one large commit.
- Amend or squash commits before creating a pull request when appropriate to maintain a clean project history.
